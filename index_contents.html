Explanation

Type a domain name to the input box and click Submit, the server will retrieve its certificate and submit its fingerprints back to the browser. You should visit the same domain with your browser and confirm that the fingerprints match. This server is most likely hosted at the IP of 54.76.1.218 with a reverse DNS look-up of ec2-​54-76-1-218.​eu-west-1.​compute.​amazon​aws.com so it will give an indipendent view on each website's HTTPS certificate.

If you are at a coffee shop, on campus or office premises, mis-matching fingerprints might indicate that the network provider is decrypting and re-encrypting all supposedly secure HTTPS connections. They can use this for scanning your browsing habbits for forbidden content, leaking of confidential information and other various purposes. This is the man-in-the-middle attack.

This site is inspired by GRC.com/fingerprints.htm which offers a similar service. See the site for more details on false positives, how to view certificates in different browsers etc.
